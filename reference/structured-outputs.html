Structured Outputs

JSON Schema–constrained model text in the Responses API.

Parameter What it does
text.format.type: "json_schema" Model output must match your schema (Structured Outputs)
text.format.strict: true Enforce exact schema adherence (use in production)
text.format.name Identifier for the schema (required, max 64 chars)
text.format.schema JSON Schema object describing allowed output shape
output_text Still a string — JSON.parse() to get an object 
text: {
  format: {
    type: "json_schema",
    name: "feature_idea",
    strict: true,
    schema: {
      type: "object",
      properties: {
        title: { type: "string" },
        priority: { type: "string", enum: ["low", "medium", "high"] },
      },
      required: ["title", "priority"],
      additionalProperties: false,
    },
  },
}
Mode When to use
json_schema (Structured Outputs) Default for gpt-4o+ and gpt-4.1 — reliable typed fields for UIs
json_object (JSON mode) Legacy — valid JSON only, no schema guarantee
text (default) Free-form prose — chat, copy, explanations

Schema rules (strict mode)

Set additionalProperties: false on objects.
List every field in required.
Use supported types: string, number, integer, boolean, enum, array, object.
Root must be type: "object" (not an array at the top level).

Frontend mapping

const data = JSON.parse(response.output_text) → set React state / Zod validate → render UI.
Pair with instructions for behavior; schema enforces shape.
Optional SDK path: client.responses.parse() + zodTextFormat for auto-parsed output_parsed — see practice/0005-structured-zod.mjs.
