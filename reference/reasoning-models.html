Reasoning Models

Thinking tokens, effort tuning, and billing — Responses API.

Fact Detail
What they do Spend hidden reasoning tokens before visible output — planning, debugging, tool use
Enable Use a reasoning-capable model + reasoning: { effort: "low" | "medium" | "high" | … }
Billing Reasoning tokens count as output tokens — not visible in output_text
Observe usage.output_tokens_details.reasoning_tokens
Summary reasoning: { summary: "auto" } → reasoning item in output
Tool loops Pass reasoning items forward (previous_response_id or full output replay)
Stateless / ZDR include: ["reasoning.encrypted_content"] to round-trip reasoning across turns 
const response = await client.responses.create({
  model: "o4-mini",           // or gpt-5.4-mini, gpt-5.5
  reasoning: { effort: "low" },
  input: hardMultiStepTask,
  store: true,
});

const reasoning = response.usage?.output_tokens_details?.reasoning_tokens;
console.log(response.output_text, { reasoning_tokens: reasoning });
Effort When
low Default try — tool use, support chat, drafting
medium Agentic coding, research, judgement calls
high / xhigh Hard debugging, deep planning — justify with evals
max_output_tokens
Reasoning consumes the output budget. Too low → status: "incomplete" with incomplete_details.reason === "max_output_tokens" — possibly zero visible text.
