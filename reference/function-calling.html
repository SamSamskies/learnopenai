Function Calling

Custom tools your code executes — Responses API.

Term What it is
tools[] Functions you declare on each request (name, description, JSON Schema parameters).
function_call Output item when the model wants your code to run. Has name, arguments (JSON string), call_id.
function_call_output Input item you send back with the tool result. Must match call_id.
tool_choice "auto" (default), "required", "none", or force one function by name.
parallel_tool_calls false → at most one tool per turn. Recommended for nano models.
strict: true Arguments must match schema (same rules as structured outputs).

Two-turn loop

// 1. Ask with tools
const r1 = await client.responses.create({ input, tools });

// 2. Find function_call in r1.output
const call = r1.output.find(i => i.type === "function_call");
const result = myHandler(JSON.parse(call.arguments));

// 3. Return output, chain thread
const r2 = await client.responses.create({
  previous_response_id: r1.id,
  input: [{ type: "function_call_output", call_id: call.call_id, output: result }],
});

// 4. User-facing text
r2.output_text
Tool type Who runs it
