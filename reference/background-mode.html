Background Mode

Async Responses API jobs — start, poll, finish without holding HTTP open.

Fact Detail
Enable background: true on responses.create
Requires store: true — stateless requests are rejected
Poll client.responses.retrieve(response.id)
Cancel client.responses.cancel(response.id)
Retention Response data kept ~10 min for polling (not ZDR-compatible) 
const started = await client.responses.create({
  model: "gpt-4.1-nano",
  input: longRunningTask,
  background: true,
  store: true,
});

let resp = started;
while (resp.status === "queued" || resp.status === "in_progress") {
  await sleep(2000);
  resp = await client.responses.retrieve(resp.id);
}

console.log(resp.status, resp.output_text);
Status Meaning
queued Waiting to start — keep polling
in_progress Model is working — keep polling
completed Done — read output_text
failed / cancelled Terminal — handle error or user abort

Sync vs background vs stream

Sync (background: false) — hold connection until done; risk HTTP timeout on long jobs
Stream (Lesson 2) — incremental tokens over one connection; UI updates live
Background — return response.id immediately; poll or notify later. Can combine background: true + stream: true to resume a dropped stream via starting_after
