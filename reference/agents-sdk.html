Agents SDK

Orchestration on top of the Responses API — TypeScript / Python.

Layer You own SDK owns
Raw Responses API Every turn, tool loop, state chain Model + hosted tools only
Agents SDK Agent defs, tool execute, product logic Tool loop, handoffs, tracing, sessions

Core API

import { Agent, run, tool } from "@openai/agents";
import { z } from "zod";

const lookup = tool({
  name: "get_feature_status",
  description: "Look up backlog by slug.",
  parameters: z.object({ feature_slug: z.string() }),
  execute: async ({ feature_slug }) => { /* your handler */ },
});

const agent = new Agent({
  name: "Product Triage",
  model: "gpt-4.1-nano",
  instructions: "Use tools when you need live data.",
  tools: [lookup],
  outputType: TriageCardSchema,  // optional Zod structured output
});

const result = await run(agent, userMessage, { maxTurns: 5 });
result.finalOutput;   // typed when outputType set
result.newItems;      // tool calls, messages, handoffs
result.lastAgent;     // which agent answered
result.history;       // pass back for turn 2

The agent loop

inspect model response
  → final output?     return result
  → handoff?          switch agent, continue
  → tool calls?       execute tools, append results, loop

Same semantics as your hand-rolled capstone loop — the SDK just runs it until finalOutput or maxTurns.

When to use which

Stay on raw Responses API Reach for Agents SDK
One or two turns you control Multi-step tool loops (3+ turns)
Custom streaming UI reducer (Lesson 16) Multiple specialist agents + handoffs
Minimal deps, wire-level learning Guardrails, approvals, tracing out of the box
Built-in tools only, one request Mix function tools + hosted tools + MCP in one agent

State strategies

Strategy Next turn you pass
result.history Full history — client-managed
session Same session instance — persistent chat
previousResponseId result.lastResponseId + new user message
conversationId Same id + new user message — OpenAI-managed

Hosted tool helpers

Same built-ins as Lesson 8 / 17 — SDK wraps them:

import {
  webSearchTool,
  fileSearchTool,
  codeInterpreterTool,
} from "@openai/agents";

tools: [
  webSearchTool({ searchContextSize: "medium" }),
  fileSearchTool(process.env.FILE_SEARCH_VECTOR_STORE_ID, { maxNumResults: 3 }),
]
