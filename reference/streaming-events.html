Streaming Events

SSE events from POST /v1/responses with stream: true.

Event type When / what to do
response.created Stream opened. Response id is available; generation not finished.
response.output_text.delta Text chunk in event.delta. Append to your UI string.
response.output_item.added Output item started — for tools: item.name, item.call_id when item.type === "function_call".
response.function_call_arguments.delta Partial tool-args JSON in event.delta. Append; don't parse until done.
response.function_call_arguments.done Final args in event.arguments. Safe to JSON.parse and run handler.
response.output_text.done One text part finalized. Full text in event.text.
response.completed Done. Full response in event.response (id, usage, output). Derive text from output or use response.output_text.done.
response.failed Error. Read event.response for details; show error in UI.

Structured JSON streaming

With text.format: json_schema, deltas are still JSON text chunks — not per-field events.
Accumulate for preview; JSON.parse only when complete.
SDK: client.responses.stream() + finalResponse() → output_parsed. 

const stream = await client.responses.create({
  model: "gpt-4.1-nano",
  input: "Your prompt",
  stream: true,
});

for await (const event of stream) {
  if (event.type === "response.output_text.delta") {
    process.stdout.write(event.delta); // UI: append to state
  }
}

UI pattern

Start → show empty / loading state
Each delta → text += event.delta (or React setText(t => t + delta))
Completed → hide spinner, store event.response.id as previous_response_id
Failed → show error, allow retry
