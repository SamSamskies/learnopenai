Instructions & Prompts

System-level guidance in the Responses API.

Mechanism What it is Persists via previous_response_id?
instructions Top-level system prompt for this request No — resend every turn
developer role in input System message inside the transcript Yes — part of stored history
user role in input End-user message Yes 
const instructions = `
You are a support bot for Acme Corp.
Answer in one short paragraph. Never invent pricing.
`.trim();

const response = await client.responses.create({
  model: "gpt-4.1-nano",
  instructions,
  input: userMessage,
  previous_response_id: lastResponseId,
  store: true,
});
Rule Detail
Priority instructions overrides conflicting content in input
Chaining previous_response_id replays user/assistant turns — not prior instructions
Chat Completions { role: "system" } → top-level instructions (or developer in input)
Product pattern Keep prompts in a versioned module; build instructions per feature; resend on every request

When to use which

instructions → stable per-feature rules you may change without polluting history (support bot tone, output format)
developer in turn 1 input → rules that should live in the stored transcript (rare; harder to update mid-thread)
Resend instructions → every turn when using previous_response_id
