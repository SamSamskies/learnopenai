Google Stitch

AI-native design canvas — prompt → iterate → export DESIGN.md → code.

What it is

Stitch (Google Labs) turns natural language and images into high-fidelity UI mockups. It is not a code editor — it is a design environment with a design agent, infinite canvas, and exports that bridge to development tools (Figma, front-end code, DESIGN.md, MCP).

Glossary

Term Meaning
Vibe design Start from business intent, user feeling, and references — not wireframes
Canvas Infinite workspace; multiple screens and context (images, text, code)
Design agent Reasons across the whole project; accepts chat and voice critique
Agent manager Parallel design directions without losing organization
DESIGN.md Agent-friendly markdown export: colors, type, spacing, components
Play mode Link screens into clickable prototype flows
Stitch MCP Connect Stitch to IDEs/agents for design context in coding sessions

Workflow (generic)

1. Intent    — who is the user, what should they feel, what job does the UI do?
2. Inventory — list regions + states (even on a single page)
3. Generate  — starter prompt → multiple screens on canvas
4. Iterate   — targeted follow-ups per screen (not full regen)
5. Prototype — optional Play mode for click-through validation
6. Export    — DESIGN.md (for agents) and/or Figma / front-end code
7. Implement — apply design system in your repo (see Lesson 35)

Prompt structure

Section Include
Product Name, audience, one-sentence job-to-be-done
Platform Web / mobile, width, light/dark
Brand feel Adjectives + anti-patterns (“no purple AI gradients”)
Chrome Persistent header, nav, composer
Screens One subsection per state with sample content
Typography Hierarchy expectations, density, whitespace

Model tips

  • Gemini 3.1 — better first-pass layout and prompt interpretation
  • Gemini 3 Flash — faster; good for quick variants
  • Nano Banana — remix / bolder visual directions via Redesign

Credits: free tier (~400/day at time of writing). New projects cost more than single edits.

Exports

Export Use when
DESIGN.md Implementing in Cursor / Claude Code with an agent
Paste to Figma Hand-off to a design team or Figma design system
Front-end code Quick static HTML/CSS starting point (still review + integrate)
Stitch MCP Pull live design context into agentic IDE sessions

Single-page apps

One route ≠ one screen. Map every phase or conditional region as a separate artboard. Persistent chrome (header, composer) must match across all of them. See Lesson 34.
