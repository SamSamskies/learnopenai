Streaming UI State

Map Responses API stream events → UI phases for capstone-shaped features.

State shape

type TriagePhase =
  | "idle"
  | "streaming-args"
  | "running-handler"
  | "streaming-card"
  | "done"
  | "error";

type TriageUIState = {
  phase: TriagePhase;
  toolName: string | null;
  argsPreview: string;
  cardPreview: string;   // raw JSON text — optional display
  card: TriageCard | null; // set only when phase → done
  error: string | null;
};

Event → state

Stream event State update
output_item.added (function_call) phase: "streaming-args", set toolName
function_call_arguments.delta append argsPreview
function_call_arguments.done replace argsPreview with final args
response.completed (turn 1) phase: "running-handler" → run handler → start turn 2
turn 2 stream starts phase: "streaming-card"
output_text.delta append cardPreview (optional)
finalResponse() card: output_parsed, phase: "done"

What to render

Phase UI
streaming-args “Calling {toolName}…” + monospace args preview
running-handler Spinner / “Looking up backlog…”
streaming-card Skeleton card or raw JSON preview
done Typed TriageCard fields — not partial JSON

Server boundary

Browser                    API route (your server)
  │                              │
  │── POST /api/triage ─────────►│ responses.create (turn 1, stream)
  │◄── SSE: ui events ───────────│ map events → { phase, toolName, … }
  │                              │ run handler
  │                              │ responses.stream (turn 2)
  │◄── SSE: card done ───────────│ output_parsed → { card }
  │                              │

Never expose OPENAI_API_KEY to the browser. The API route owns the two-turn loop from capstone pattern.
