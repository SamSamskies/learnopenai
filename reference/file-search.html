File Search

Hosted RAG — vector store + file_search built-in tool.

Setup pipeline

files.create(purpose: "assistants")
  → vectorStores.create()
  → vectorStores.files.create(vector_store_id, file_id)
  → poll until status === "completed"
  → responses.create({ tools: [{ type: "file_search", vector_store_ids }] })
Step API Notes
Upload client.files.create purpose: "assistants"
Store client.vectorStores.create Reusable knowledge base id
Attach vectorStores.files.create Poll until indexed
Query responses.create + file_search One request — no turn 2

Tool config

tools: [{
  type: "file_search",
  vector_store_ids: ["vs_…"],
  max_num_results: 3,        // fewer = cheaper/faster
  filters: { … },            // metadata on vector store files
}]

Response shape

Output item Contains
file_search_call status, queries used for retrieval
message output_text + file_citation annotations 
const call = response.output.find(i => i.type === "file_search_call");
const msg = response.output.find(i => i.type === "message");

call?.queries;
response.output_text;
msg?.content[0].annotations;  // [{ type: "file_citation", filename, file_id, index }]

Pricing

Component Price Notes
Storage $0.10 / GB / day First 1 GB free
Tool calls $2.50 / 1k calls Per file_search_call
Model tokens Model rate Question + retrieved chunks + answer

Compare: web_search is ~$10 / 1k calls — file search is cheaper per call but adds storage. Custom RAG can win at scale; hosted wins when you want zero retrieval infra. Source: OpenAI pricing.

Hosted RAG across providers

“Hosted RAG” = upload docs, ask questions, get citations — minimal retrieval code. Vector-DB free tiers (Supabase, Pinecone, Qdrant) are cheaper at scale but you wire chunking, embeddings, and the LLM yourself.

Option Free tier highlights Tradeoff
OpenAI file_search 1 GB storage free; fine for practice $0.10/GB/day storage + $2.50/1k calls; stays in Responses API
Google File Search 1 GB storage; indexing free on free tier No per-search fee; different API (generateContent)
Supabase pgvector 500 MB DB with vectors DIY retrieval loop; project pauses after 1 week idle
Pinecone Starter 2 GB vector storage + read/write quotas Vector DB only — add embeddings + LLM
Qdrant Cloud ~1 GB permanent free tier Same — retrieval infra, not end-to-end RAG
Anthropic Files API (attach docs to messages) No hosted semantic search over a corpus 
Hosted doc Q&A, minimal code     → Gemini File Search (generous free tier)
Already on Responses API stack   → OpenAI file_search (simplest integration)
Max free experimentation + code OK → Supabase pgvector or Pinecone/Qdrant + your embeddings

Sources: Gemini File Search · Gemini pricing · Supabase pricing · Pinecone pricing · Claude Files API. See also RESOURCES.md — Hosted RAG section.

Model choice

Platform handles retrieval. Model handles query formulation, reading chunks, answering. Default: gpt-4.1-mini.

Mini enough Step up
Structured docs, factual lookups Dense specs, contracts, legal
Single-source, clear answers Multi-hop across many files
User verifies via citations High-stakes faithfulness to source 
Start gpt-4.1-mini → test real docs + questions
  → miss nuance / wrong sections / mash sources?
  → bump model before rebuilding RAG
  → tune max_num_results and doc quality first
