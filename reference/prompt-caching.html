Prompt Caching

Automatic prefix reuse for cheaper, faster repeated prompts.

Fact Detail
Enabled by default No opt-in. Works on gpt-4o and newer models.
Minimum size Prefix must be ≥ 1024 tokens for cache writes/reads.
Match rule Exact prefix match — byte-for-byte identical start of prompt.
What counts in prefix instructions, tools, prior chained turns, images, structured-output schema.
Observe hits usage.input_tokens_details.cached_tokens
Retention In-memory ~5–10 min idle (up to 1 h). Extended 24h on select models (e.g. gpt-4.1). 
const response = await client.responses.create({
  model: "gpt-4.1-nano",
  instructions: stableLongInstructions,  // static — front of prompt
  input: userMessage,                    // variable — end of prompt
  tools,
  store: true,
});

const cached = response.usage?.input_tokens_details?.cached_tokens ?? 0;
const input = response.usage?.input_tokens ?? 0;

Uncached input $0.10

Product prompt layout

Front (static) → instructions, tool defs, few-shot examples, RAG context that rarely changes
End (dynamic) → user message, per-session variables, timestamps
Chained threads → prior turns append to prefix; resending identical instructions each turn still caches that block

Cache breaker Example
Editing instructions New body = new prefix. Tiny edits in a huge repeated block may still partially cache.
Reordering tools tools array must be identical.
Injecting timestamps in system prompt Put dynamic values in input, not instructions.
High burst rate > ~15 req/min per prefix may overflow to new machines.
