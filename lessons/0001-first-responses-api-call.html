Lesson 1 · ~15 minutes

Your First Responses API Call

ChatGPT is a product. The Responses API is the platform. AI Product Engineers ship on the platform.

Your mission is to build products on top of LLM APIs — not just chat in a UI. Everything starts with one authenticated call to POST /v1/responses and understanding what comes back.

Two products, one company

Product URL You pay for
ChatGPT chatgpt.com Go / Plus / Pro subscription
OpenAI Platform platform.openai.com Pay-as-you-go API credits
Not the same bill
A ChatGPT subscription does not fund API calls. For this course you need a platform.openai.com account with billing set up (~$5 minimum prepaid credits).

Setup (one time)

Step 1 Create an account at platform.openai.com.
Step 2 Go to Settings → Billing, add a payment method, and purchase at least $5 in credits. Set a monthly spend limit while learning.
Step 3 Create an API key at API Keys. Copy it once — you won't see it again.
Step 4 Export it in your terminal (add to ~/.zshrc to persist): 
export OPENAI_API_KEY="sk-..."

The official SDKs read OPENAI_API_KEY automatically. Never commit this key to git or paste it into frontend code.

The call

The Responses API is OpenAI's current unified surface. One endpoint handles text generation, tools, images, and server-side state. Docs. 

import OpenAI from "openai";

const client = new OpenAI();

const response = await client.responses.create({
  model: "gpt-4.1-nano",
  input: "Say hello in exactly five words.",
});

console.log(response.output_text);

gpt-4.1-nano is cheap for learning (~$0.10 per million input tokens). A few hundred practice calls cost pennies.

Read the response object

Don't assume a flat string. A response is a structured object:

Property What to notice
output_text SDK convenience — the model's text. Start here.
output Array of items: messages, tool calls, reasoning blocks. You'll parse this when you add tools.
id resp_… — you'll pass this back as previous_response_id for multi-turn conversations.
usage Token counts. Watch these as you build — they are your unit economics.
status completed, in_progress, failed. Matters for streaming and background mode later.

Keep the Responses API reference open while you work.

Do this now

10-minute win: run a real API call from your machine.

  1. In this repo, open a terminal at the project root
  2. Install the SDK: npm init -y && npm install openai
  3. Confirm your key is set: echo $OPENAI_API_KEY | head -c 8 (should print sk-proj- or similar)
  4. Run the practice script: node practice/0001-hello.mjs
  5. Log the full response once: add console.log(JSON.stringify(response, null, 2)) and find output, usage, and id

You should see five words of greeting text, a resp_… id, and token counts. That is a working integration — everything else in this course builds on it.
