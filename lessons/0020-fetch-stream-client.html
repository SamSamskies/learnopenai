Fetch Stream Client

Same SSE on the wire. A client React can own.

Lesson 19 proved the server boundary with EventSource. That works for demos. In a Next.js app you usually switch to fetch + a stream reader — same data: {json}\n\n format, but POST bodies, auth headers, and AbortController for React lifecycle.

Why not EventSource in production?

Need EventSource fetch stream
Send the user's question in the body GET only POST with JSON body
Attach session / auth headers Not supported Standard headers
Cancel when user clicks again source.close() abortController.abort()
Wire format from Lesson 19 Identical — UI snapshots, not raw OpenAI events

See fetch stream client reference and streaming UI state.

The parse loop

Chunks can split mid-line. Buffer until you see a full \n\n frame, then parse each data: line: 

const reader = res.body.getReader();
const decoder = new TextDecoder();
let buffer = "";

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buffer += decoder.decode(value, { stream: true });
  const frames = buffer.split("\n\n");
  buffer = frames.pop() ?? "";
  for (const frame of frames) {
    const line = frame.trim();
    if (!line.startsWith("data: ")) continue;
    const state = JSON.parse(line.slice(6));
    dispatch(state);  // useReducer in React
  }
}

React mapping

The server still runs the two-turn loop from Lesson 15. The client reducer stays dumb — it merges snapshots, it does not interpret OpenAI events: 

// components/TriagePanel.tsx (sketch)
const [state, dispatch] = useReducer(
  (s, patch) => ({ ...s, ...patch }),
  initialTriageState
);

// Hook wraps fetch + parse loop; calls dispatch(snapshot) per frame
const { start } = useTriageStream(dispatch);

// Render: switch on state.phase — same branches as triage-demo.html

In Next.js, the route handler returns the same stream Lesson 19 wrote by hand — wrapped in new Response(stream, { headers: { "Content-Type": "text/event-stream" } }). Set export const runtime = "nodejs" — the OpenAI SDK needs Node, not Edge.

Vercel AI SDK
Many Next.js apps use the ai package (useChat, streamText) instead of hand-rolling this loop. Same idea: server streams, client consumes. Learn the raw pattern first — then you'll know what the library abstracts.
