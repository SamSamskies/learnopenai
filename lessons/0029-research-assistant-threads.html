Quick reference: conversation state · research assistant.

Research vs triage: one id per turn

Triage sometimes needs two responses per user message (tool turn, then structured card). You saved turn 2's id. Research is simpler — one responses.stream() call per user message (web search + brief in a single response). Save that response's id.

Capstone Responses per user message Save this id
Triage (backlog path) Turn 1: function_call · Turn 2: card Turn 2 response.id
Research Assistant Single stream: search (optional) + ResearchBrief finalResponse().id

Resend every turn
instructions and tools do not carry over the chain. Your streamResearch helper already sends them — keep doing so on every turn. Only the new user input and previous_response_id change.

1. Session map — lib/sessions.ts

In-memory Map for this lesson — same as practice/0023-triage-conversation.mjs. In production, persist lastResponseId in your DB next to the thread row. 

const sessions = new Map<string, string>();

export function getLastResponseId(sessionId: string): string | null {
  return sessions.get(sessionId) ?? null;
}

export function setLastResponseId(sessionId: string, responseId: string) {
  sessions.set(sessionId, responseId);
}

export function clearSession(sessionId: string) {
  sessions.delete(sessionId);
}

2. Stream helper — chain + return id

Add store: true, optional previous_response_id, and return response.id so the route can save it. 

export async function streamResearch(
  message: string,
  onSnapshot: (state: ResearchUIState) => void,
  { previousResponseId }: { previousResponseId?: string } = {}
): Promise<string | null> {
  const stream = openai.responses.stream({
    model: "gpt-5-mini",
    instructions: "…",
    input: message,
    tools: [{ type: "web_search", search_context_size: "low" }],
    text: { format: zodTextFormat(ResearchBrief, "research_brief") },
    store: true,
    ...(previousResponseId && { previous_response_id: previousResponseId }),
  });

  // … same event loop as Lesson 28 …

  const response = await stream.finalResponse();
  // … set brief, citations, phase done …
  onSnapshot({ ...state });
  return response.id;
}

Optionally add responseId: string | null to ResearchUIState and include it in the final snapshot — useful for debugging, same as triage's responseId field.

4. New chat — DELETE /api/session

A New chat button should do three things:

  1. DELETE /api/session?id=<sessionId> — clear server map entry
  2. Issue a fresh UUID in sessionStorage
  3. Clear the client thread array
// app/api/session/route.ts
export async function DELETE(req: Request) {
  const sessionId = new URL(req.url).searchParams.get("id");
  if (sessionId) clearSession(sessionId);
  return new Response(null, { status: 204 });
}
