Lesson 6 · ~15 minutes

Stream Structured Output

Fast UI feedback. Typed fields at the end.

Lesson 2 streamed prose. Lesson 5 returned typed JSON. Real AI products combine both: the user sees something arrive immediately, then your UI renders a structured card when generation finishes.

What actually streams

With text.format: json_schema (or Zod), deltas are still raw JSON text chunks — not individual fields. You get {"tit then le":"Dark then Mode"... the same response.output_text.delta event from Lesson 2.

Don't parse partial JSON
JSON.parse on incomplete deltas will throw. Accumulate the string for a live preview if you want, but only validate and render typed fields when the stream completes.

The combined pattern

const stream = client.responses.stream({
  model: "gpt-4.1-nano",
  instructions: "Extract a feature idea from feedback.",
  input: messyUserMessage,
  text: { format: zodTextFormat(FeatureIdea, "feature_idea") },
});

let jsonBuffer = "";

for await (const event of stream) {
  if (event.type === "response.output_text.delta") {
    jsonBuffer += event.delta;       // optional: show raw preview
    process.stdout.write(event.delta);
  }
}

const response = await stream.finalResponse();
// response.output_parsed — Zod-validated, safe for React state
renderCard(response.output_parsed);
Phase UI state
Stream opens Skeleton / spinner on the card
Each delta Optional monospace JSON preview (append deltas)
finalResponse() Parse once → render title, priority, etc.
response.failed Error state, retry button
Frontend mapping
During stream: setPreview(prev => prev + event.delta) or skip preview entirely.
On complete: setData(response.output_parsed) — swap skeleton for the real card.
Store response.id from finalResponse() if chaining.

SDK methods recap

Goal Method
Blocking typed output client.responses.parse() → output_parsed
Streaming typed output client.responses.stream() → deltas, then finalResponse()
Wire primitive (any language) create({ stream: true, text: { format: json_schema } })

See structured outputs reference for create vs parse vs stream, and streaming events reference for event types.
