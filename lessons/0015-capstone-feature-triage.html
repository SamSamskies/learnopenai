Capstone: Feature Triage

Fourteen lessons of parts. One product-shaped flow.

Your mission success criteria include shipping a small feature with streaming UI, tool calls, and structured JSON output. This lesson wires all three into one flow — the same shape most AI product surfaces use.

The feature

A PM asks: “Users keep asking for dark mode — is it on our roadmap?” Your assistant should:

  1. Stream a tool call to your backlog API (Lesson 9)
  2. Run your handler with real data (Lesson 7)
  3. Stream a structured triage card the UI can render (Lesson 6)
  4. Chain turns with previous_response_id (Lesson 10)

See capstone pattern reference.

Architecture

Phase What happens User sees
1 — stream tool call Model picks get_feature_status; args stream in “Calling get_feature_status…” + live JSON args
Between Your code queries the backlog (mock DB in practice) Short “looking up…” state
2 — stream structured card Model returns Zod-validated TriageCard JSON Raw JSON preview → typed card fields at end

Turn 2 — the new combination

You have streamed tool calls (Lesson 9) and streamed structured output (Lesson 6) separately. Turn 2 combines both and chains state: 

const turn2 = client.responses.stream({
  model: "gpt-4.1-nano",
  previous_response_id: turn1ResponseId,
  instructions: "Return a triage card grounded in the tool result.",
  input: [{
    type: "function_call_output",
    call_id: callId,
    output: toolResult,
  }],
  text: { format: zodTextFormat(TriageCard, "triage_card") },
});

for await (const event of turn2) {
  if (event.type === "response.output_text.delta") {
    jsonBuffer += event.delta; // optional live preview
  }
}

const card = (await turn2.finalResponse()).output_parsed;
// → { feature_title, current_status, priority, recommendation }
Still don't parse partial JSON
Deltas are raw JSON text. Accumulate for preview; render typed fields only after finalResponse() — same rule as Lesson 6.

What you already know — when to layer it in

Need Add from
OpenAI runs web search / files Lesson 8 — one request, no handler
External vendor's tool server Lesson 14 — type: "mcp"
Cheap model fails evals Lesson 13 — upgrade + measure
Big repeated system prompt Lesson 11
Job outlives HTTP timeout Lesson 12

The capstone uses custom functions + nano on purpose — you own the handler and the schema. Production features pick the right layer per capability, not everything at once.

Frontend mapping

In React (or any UI), each phase maps to state: 

toolPhase: "idle" | "streaming-args" | "running-handler" | "streaming-card" | "done"
toolName: string | null
argsPreview: string          // from function_call_arguments.delta
card: TriageCard | null       // set once after finalResponse()
