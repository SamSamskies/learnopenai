AI SDK Server Migration

Same SSE frames for the client. Different engine under the route.

Lesson 31 shipped a guarded, testable Research Assistant. The streaming loop in lib/stream-research.ts still called openai.responses.stream() directly. This lesson swaps the server to Vercel AI SDK Core — streamText + openai.responses() — while keeping the exact ResearchUIState contract your client and Vitest already expect.

What changes (and what doesn’t)

Unchanged Migrated
POST /api/research SSE snapshots openai.responses.stream() → streamText()
ResearchChat.tsx client zodTextFormat → Output.object({ schema })
Guards, sessions, upload route Event loop: raw OpenAI events → AI SDK stream parts
Vitest SSE contract tests Mock target: streamText instead of openai.responses.stream

Event mapping

AI SDK normalizes provider streams into typed parts. Map them to your existing phase machine:

AI SDK stream part ResearchUIState
tool-call · toolName === "web_search" phase: "searching", searched: true
tool-call · toolName === "file_search" phase: "searching", searchedDocs: true
text-delta phase: "streaming-answer", append briefPreview
After stream: await result.output brief (parsed Zod object)
After stream: text content annotations sources (url + file citations)
finalStep.providerMetadata.openai.responseId Return for previous_response_id threading

Core call shape

import { openai } from "@ai-sdk/openai";
import { Output, streamText } from "ai";
import { ResearchBrief } from "@/lib/schemas";

const result = streamText({
  model: openai.responses("gpt-5-mini"),
  system: INSTRUCTIONS,
  prompt: message,
  tools: {
    web_search: openai.tools.webSearch({ searchContextSize: "low" }),
  },
  output: Output.object({ schema: ResearchBrief, name: "research_brief" }),
  providerOptions: {
    openai: {
      store: true,
      previousResponseId, // optional — thread continuity
    },
  },
});

for await (const part of result.stream) {
  // map tool-call + text-delta → onSnapshot(state)
}

const brief = await result.output;
const responseId = (await result.finalStep).providerMetadata?.openai?.responseId;
