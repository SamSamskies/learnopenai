Design to Code

DESIGN.md is the spec. ResearchChat.tsx is the behavior.

Lesson 34 exported research-assistant/DESIGN.md from Stitch — colors, typography, spacing, and component rules for the "Scholarly Precision" identity. This lesson applies that design system to ResearchChat.tsx without changing any product logic.

Architecture after restyle

DESIGN.md (Stitch export)
    │
    ├─► :root CSS variables     (--primary, --outline-variant, …)
    ├─► @theme inline           (Tailwind color/font/radius tokens)
    ├─► layout.tsx              (next/font: Geist + Source Serif 4)
    └─► ResearchChat.tsx        (semantic classes, subcomponents)

Step 1 — Map tokens to CSS variables

Open research-assistant/DESIGN.md and extract the colors you will use in components. Add them to app/globals.css: 

:root {
  --background: #ffffff;
  --foreground: #191c1d;
  --primary: #0f4c5c;
  --primary-dark: #003441;
  --outline-variant: #c0c8cb;
  --surface-container-low: #f3f4f5;
  /* … error tokens, on-primary, etc. */
}

@theme inline {
  --color-primary: var(--primary);
  --color-outline-variant: var(--outline-variant);
  --font-serif: var(--font-source-serif);
  /* … wire every var you need as a Tailwind token */
}

Rule: components use semantic Tailwind classes (bg-primary, border-outline-variant) — never raw hex in JSX.

Step 2 — Load fonts in layout

DESIGN.md specifies Source Serif 4 for headlines and Geist for UI. In app/layout.tsx: 

import { Geist, Geist_Mono } from "next/font/google";
import { Source_Serif_4 } from "next/font/google";

const sourceSerif = Source_Serif_4({
  variable: "--font-source-serif",
  subsets: ["latin"],
  weight: ["400", "500", "600"],
});

Apply font variables on <html>. Add a utility in globals.css: 

.font-serif {
  font-family: var(--font-source-serif), Georgia, serif;
}

Step 4 — Prompt Cursor (or restyle by hand)

If using an agent, point it at both files and constrain the scope: 

Restyle ResearchChat.tsx against research-assistant/DESIGN.md.
Do NOT change useChat, API calls, phase logic, or tests.
Map DESIGN.md colors to globals.css CSS variables.
Use font-serif for brief headlines and empty state.
Use borders instead of shadows. Verify all ResearchUIState phases render.

Step 5 — Verify all phases

Run npm run dev and walk through every UI state:

  1. Empty — serif headline, no thread yet
  2. Searching — spinner + skeleton card
  3. Streaming — collapsible brief preview panel
  4. Done — brief card with headline, points, source pills, grounding badges
  5. Error — error banner + retry button
  6. Upload — file chips above composer
  7. Multi-turn — older turns still render correctly

Then run npm test and npm run build. Tests should pass unchanged — you only touched presentation.
