Platform skills are done. Now you ship a product.

Lesson 24 closed the OpenAI primitives list. You already proved the hard parts in practice — streaming SSE, tool calls, structured output, file search, guards, multi-turn state. This lesson starts the capstone app: a Next.js research assistant that reuses those patterns in a stack you know.

What you're building

A research assistant for solo builders: ask a question, get a web-grounded answer with citations, follow up in the same thread, optionally upload docs for grounded answers. Not another triage demo — a product you'd actually use.

Full roadmap: research assistant reference. This lesson only scaffolds the shell; each following lesson adds one slice.

Architecture (end state)

Browser (React)          Next.js route handler          OpenAI
     │  POST /api/research              │
     │  { message, sessionId }          │
     │─────────────────────────────────►│ guard → stream loop
     │                                  │ web_search / file_search
     │◄── SSE UI snapshots ─────────────│ store + previous_response_id
     │  phase: searching | streaming    │
     │  phase: done + citations         │

Same server boundary as Lesson 19 — API key never in the browser. Same SSE contract as Lesson 20 — UI phase snapshots, not raw OpenAI events.

Do this now

25-minute win: a running Next.js shell named Research Assistant.

  1. From the repo root, scaffold the app: 
    cd /path/to/learnopenai
npx create-next-app@latest research-assistant \
  --typescript --eslint --app --no-src-dir \
  --import-alias "@/*" --turbopack --yes
    Skip Tailwind if you prefer — add --tailwind or accept the default prompt.
  2. Install the OpenAI SDK: 
    cd research-assistant
npm install openai zod
  3. Copy env (server-only): 
    echo "OPENAI_API_KEY=sk-..." > .env.local
    Use the same key as your root .env. Never commit .env.local.
  4. Edit app/page.tsx — replace the default page with a minimal shell:
    • Title: Research Assistant
    • Subtitle: Web-grounded research for solo builders
    • Placeholder textarea + disabled Send button (wired in Lesson 27)
  5. Run npm run dev — confirm http://localhost:3000 shows your shell
  6. Skim research assistant reference — note the lesson arc through Lesson 31
  7. Optional: clone OpenAI Responses starter app in a temp folder and compare its folder layout to yours

Next lesson (26): add app/api/research/route.ts — a non-streaming POST that calls web_search and returns answer + citations. You'll curl it before building the React client.
