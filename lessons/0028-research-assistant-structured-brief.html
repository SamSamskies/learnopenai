Lesson 28 · ~30 minutes

Structured Research Brief

Plain prose is for chat. A schema is for your product card.

Lesson 27 streamed a free-text answer. That works — but a research product needs predictable fields: headline, summary, bullets, confidence. Same move you made in the triage capstone with TriageCard ( Lesson 15) and practiced in Lesson 6. This lesson adds a Zod ResearchBrief and renders a typed card at done.

Architecture

React client                  stream-research.ts           OpenAI
  │                                  │                        │
  │◄── { phase, briefPreview } ──────│◄── output_text.delta ──│ web_search
  │    (raw JSON chunks)             │    (JSON text)         │ → message
  │◄── { phase: done, brief, … } ────│ finalResponse()        │
  │    typed card + citations        │ output_parsed + annot. │

The phase machine is unchanged — idle → searching → streaming-answer → done. What streams during streaming-answer is now raw JSON text, not prose. Do not JSON.parse partial briefPreview — wait for finalResponse().

Citations stay separate from the schema.
ResearchBrief holds your product fields. Citation links still come from url_citation annotations on the final message — same rule as Lessons 26–27. Never put URLs in the Zod schema and trust them; annotations are the source of truth.

1. Schema — lib/schemas.ts

Client-safe module — no OpenAI imports. Export both the Zod schema and the inferred TypeScript type. 

import { z } from "zod";

export const ResearchBrief = z.object({
  headline: z
    .string()
    .describe("One-line takeaway for a busy builder"),
  summary: z
    .string()
    .describe("Two or three sentences answering the question"),
  key_points: z
    .array(z.string())
    .describe("Three to five concise facts or implications"),
  confidence: z
    .enum(["low", "medium", "high"])
    .describe("How well the available sources support this answer"),
});

export type ResearchBrief = z.infer<typeof ResearchBrief>;

2. UI state — update lib/research-state.ts

Replace prose fields with brief fields. Import the type from schemas.ts. 

import type { ResearchBrief } from "@/lib/schemas";

export type ResearchUIState = {
  phase: ResearchPhase;
  briefPreview: string;
  brief: ResearchBrief | null;
  citations: Citation[];
  searched: boolean;
  error: string | null;
};

export function createResearchState(
  overrides: Partial<ResearchUIState> = {}
): ResearchUIState {
  return {
    phase: "idle",
    briefPreview: "",
    brief: null,
    citations: [],
    searched: false,
    error: null,
    ...overrides,
  };
}

3. Stream helper — switch to responses.stream()

responses.create({ stream: true }) does not give you output_parsed. Use responses.stream() with zodTextFormat — same pattern as turn 2 in practice/lib/stream-triage.mjs, but in a single turn with web_search. 

import { zodTextFormat } from "openai/helpers/zod";
import { ResearchBrief } from "@/lib/schemas";

export async function streamResearch(
  message: string,
  onSnapshot: (state: ResearchUIState) => void
) {
  const state = createResearchState({ phase: "searching" });
  onSnapshot(state);

  const stream = openai.responses.stream({
    model: "gpt-5-mini",
    instructions:
      "You are a research assistant for solo builders. Search the web when the question needs current or factual grounding. Return a structured brief grounded in what you find.",
    input: message,
    tools: [{ type: "web_search", search_context_size: "low" }],
    text: { format: zodTextFormat(ResearchBrief, "research_brief") },
  });

  for await (const event of stream) {
    if (
      event.type === "response.output_item.added" &&
      event.item.type === "web_search_call"
    ) {
      state.searched = true;
      state.phase = "searching";
      onSnapshot({ ...state });
    }

    if (
      event.type === "response.output_item.added" &&
      event.item.type === "message"
    ) {
      state.briefPreview = "";
    }

    if (event.type === "response.output_text.delta") {
      state.phase = "streaming-answer";
      state.briefPreview += event.delta;
      onSnapshot({ ...state });
    }
  }

  const response = await stream.finalResponse();
  state.brief = response.output_parsed ?? null;
  state.citations = extractUrlCitations(response);
  state.searched = response.output.some(
    (item) => item.type === "web_search_call"
  );
  state.phase = "done";
  onSnapshot({ ...state });
}
finalResponse() after the loop.
Call it once the async iterator finishes — not on every response.completed event mid-loop. That is when output_parsed is Zod-validated and safe for React state.

The route handler (app/api/research/route.ts) stays the same — it still sends full ResearchUIState snapshots over SSE.
