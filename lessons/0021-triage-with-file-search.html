Triage with File Search

Live backlog or uploaded docs. One route — model picks.

Lesson 15 looked up a live backlog with a custom function. Lesson 17 answered from uploaded docs with file_search. Lesson 20 shipped the client. Real triage assistants need both — and the model should pick the right tool per question.

Two paths, one route

User asks about… Tool Turns UI output
Live backlog / current status get_feature_status Two — handler + structured card TriageCard fields
Uploaded PRD / roadmap doc file_search One — platform runs retrieval Free text + file_citation filenames

Register both in the same tools array. Write instructions that name when to use each. The server branches on the first output_item.added — same SSE contract, different phase machines per path. 

tools: [
  zodResponsesFunction({ name: "get_feature_status", … }),
  { type: "file_search", vector_store_ids: [storeId], max_num_results: 3 },
]

See streaming UI state, file search reference, and capstone pattern.

Extended phase machine

The backlog path from Lesson 16 is unchanged. The docs path adds three phases: 

routing → searching-docs → streaming-answer → done
backlog: routing → streaming-args → running-handler → streaming-card → done
Phase Path User sees
routing both “Choosing tool…” before first item
searching-docs docs “Searching documents…” (+ queries when available)
streaming-answer docs Answer text streaming in
done docs Final answer + source filenames from citations
UI state shape
Add path: "backlog" | "docs" so the client reducer knows which branches to render. Docs path uses answerPreview, answer, and citations[] instead of card. Still one snapshot per SSE frame — the client stays dumb.

Server branch (sketch)

for await (const event of turn1) {
  if (event.item?.type === "function_call") {
    path = "backlog";
    state.phase = "streaming-args";
    // … existing two-turn loop from stream-triage.mjs
  }
  if (event.item?.type === "file_search_call") {
    path = "docs";
    state.phase = "searching-docs";
  }
  if (path === "docs" && event.type === "response.output_text.delta") {
    state.phase = "streaming-answer";
    state.answerPreview += event.delta;
  }
  if (path === "docs" && event.type === "response.completed") {
    state.citations = extractFileCitations(event.response);
    state.phase = "done";  // no turn 2
  }
}
Vector store setup
file_search needs FILE_SEARCH_VECTOR_STORE_ID in .env (from Lesson 17). First run uploads product-roadmap.md and prints the id. Reuse it — a tiny doc stays under the free storage tier.
