Lesson 7 · ~15 minutes

Custom Function Tools

The model decides what to do. Your code does it.

Lesson 5 constrained what the model returns. Now the model can request actions — look up a user, check inventory, file a ticket — and your backend runs them. This is function calling (tool calling). Docs.

Three pieces of vocabulary

Term Who owns it
Tool — the function definition you declare Your API request (tools array)
Tool call — model asks to run one Model response (function_call in output)
Tool output — result your code produced Your next request (function_call_output)

The model never touches your database. It only emits a structured request; you execute it and send the result back.

Define a function tool

Same JSON Schema rules as structured outputs: strict: true, every property in required, additionalProperties: false. See structured outputs reference. 

tools: [{
  type: "function",
  name: "get_feature_status",
  description: "Look up a product feature in the backlog by slug.",
  strict: true,
  parameters: {
    type: "object",
    properties: {
      feature_slug: { type: "string", description: "e.g. dark-mode" },
    },
    required: ["feature_slug"],
    additionalProperties: false,
  },
}]

The two-turn loop

// Turn 1 — user asks; model may call your tool
const first = await client.responses.create({
  model: "gpt-4.1-nano",
  instructions: "Use get_feature_status for real backlog data.",
  input: "Is dark mode on our roadmap?",
  tools,
});

const call = first.output.find(i => i.type === "function_call");
// call.name → "get_feature_status"
// call.arguments → '{"feature_slug":"dark-mode"}'
// call.call_id → pair output to this call

const result = getFeatureStatus(JSON.parse(call.arguments));

// Turn 2 — send tool output back, chain the thread
const second = await client.responses.create({
  model: "gpt-4.1-nano",
  previous_response_id: first.id,
  input: [{
    type: "function_call_output",
    call_id: call.call_id,
    output: result,  // string (often JSON)
  }],
});

console.log(second.output_text);
// "Dark mode is planned with high priority."
Frontend mapping
Turn 1 response has no user-facing text yet — show "Looking up backlog…"
Your server runs the handler, then calls the API again with the output.
Turn 2 output_text is what you stream or render in the chat bubble.
Chain with previous_response_id from Lesson 3.
call_id must match
Every function_call_output references the call_id from its function_call. Wrong id → the model can't pair result to request (breaks with parallel calls too).

Custom vs built-in vs MCP

Tool type Runs where Example
Custom function Your server get_feature_status, refund order
Built-in OpenAI platform Web search, code interpreter
Remote MCP External MCP server Connect Cursor-style tools to the API

This lesson is custom functions only — you own the handler. Built-in tools and MCP come later.
