Lesson 5 · ~15 minutes

Structured JSON Output

Free text is for humans. JSON Schema is for your product.

In Lesson 4 you set behavior with instructions. Now constrain shape: tell the model exactly what fields to return so your UI can parse the response without regex or hope.

This is Structured Outputs — the Responses API parameter is text.format with type: "json_schema". Docs.

Why products need this

Imagine a feedback inbox. The model reads messy user text and your UI needs title, user_problem, and priority to render a card. Without a schema, the model might return prose, bullet lists, or JSON with the wrong keys — every render path breaks differently.

With Structured Outputs, the API guarantees output matches your schema (when strict: true). You JSON.parse(output_text) once and map fields to React state.

The API shape

const response = await client.responses.create({
  model: "gpt-4.1-nano",
  instructions: "Extract a feature idea from user feedback.",
  input: messyUserMessage,
  text: {
    format: {
      type: "json_schema",
      name: "feature_idea",
      strict: true,
      schema: {
        type: "object",
        properties: {
          title: { type: "string" },
          user_problem: { type: "string" },
          priority: {
            type: "string",
            enum: ["low", "medium", "high"],
          },
        },
        required: ["title", "user_problem", "priority"],
        additionalProperties: false,
      },
    },
  },
});

const data = JSON.parse(response.output_text);
// data.title, data.priority — ready for your UI
Piece Role
instructions How to think (tone, domain rules)
input What to extract from (user content)
text.format.schema What shape to return (fields and types)
output_text JSON string — parse it client-side
instructions + schema work together
Instructions say what to extract and how to phrase it. Schema says which fields must exist. Don't put field lists only in instructions — the schema is the contract your code depends on.

strict: true in production

Always set strict: true when shipping. The model must follow your schema exactly. In strict mode:

  • Root schema must be type: "object"
  • Every property needs a type
  • All fields go in required
  • Objects need additionalProperties: false
Not the same as JSON mode
Older type: "json_object" only promises valid JSON — any keys, any shape. json_schema promises your shape. Use json_schema for product UIs.

Chaining note

Like instructions, text.format applies per request. If turn 1 is free text and turn 2 needs structured JSON, set text.format on turn 2 (and resend instructions). See the structured outputs reference.

Optional: SDK auto-parse

If you add Zod, the SDK can parse for you: client.responses.parse() with zodTextFormat returns output_parsed directly. The raw text.format + JSON.parse path you learn here is the platform primitive underneath.
